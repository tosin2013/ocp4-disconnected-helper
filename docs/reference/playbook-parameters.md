# Playbook Parameter Reference

Complete reference for all playbook parameters across the project.

---

## Core Infrastructure Playbooks

### site.yml (Main Entrypoint)

**Purpose**: Deploy entire stack in correct order

**Parameters**: None (uses inventory vars)

**Tags**:
- `provision` - VM provisioning only
- `configure` - Configuration only (skip provisioning)
- `registry` - Registry infrastructure only
- `aap` - AAP deployment only
- `certificates` - Certificate setup only
- `haproxy` - HAProxy configuration only
- `validate` - Validation checks only

**Example**:
```bash
ansible-playbook -i inventory/ibm-cloud.yml playbooks/site.yml \
  --tags registry,certificates
```

### deploy-registry.yml

**Purpose**: Deploy container registry infrastructure

**Parameters**:
| Parameter | Type | Default | Required | Description |
|-----------|------|---------|----------|-------------|
| `registry_type` | string | `quay` | No | Registry type: quay, harbor, jfrog |
| `registry_fqdn` | string | `registry.example.com` | Yes | Fully qualified domain name |
| `registry_port` | int | `8443` | No | Registry HTTPS port |
| `registry_storage_path` | string | `/opt/mirror-registry/quay-storage` | No | Storage directory |
| `registry_admin_password` | string | - | Yes | Admin password (vault encrypted) |

**Example**:
```bash
ansible-playbook playbooks/deploy-registry.yml \
  -e registry_type=quay \
  -e registry_fqdn=registry.example.com \
  -e @extra_vars/rhel-subscription-secrets.yml \
  --vault-password-file ~/.vault_pass
```

### setup-mirror-registry.yml (Quay)

**Purpose**: Install Quay mirror-registry v2

**Parameters**:
| Parameter | Type | Default | Required | Description |
|-----------|------|---------|----------|-------------|
| `mirror_registry_url` | string | Auto-detected | No | Download URL for mirror-registry tarball |
| `quay_admin_username` | string | `init` | No | Initial admin username |
| `quay_admin_password` | string | - | Yes | Initial admin password |
| `quay_storage_path` | string | `/opt/mirror-registry/quay-storage` | No | Quay storage directory |
| `ssl_cert_path` | string | - | Yes | Path to SSL certificate (generated by setup-certificates.yml) |
| `ssl_key_path` | string | - | Yes | Path to SSL private key |

**Example**:

```bash
  -e quay_admin_password=SecurePassword123! \
  -e ssl_cert_path=/opt/certificates/registry.example.com.crt \
  -e ssl_key_path=/opt/certificates/registry.example.com.key
```

### setup-certificates.yml

**Purpose**: Generate or obtain SSL/TLS certificates

**Parameters**:
| Parameter | Type | Default | Required | Description |
|-----------|------|---------|----------|-------------|
| `certificate_mode` | string | `selfsigned` | No | Certificate source: selfsigned, letsencrypt |
| `certificate_fqdn` | string | `registry.example.com` | Yes | Domain name for certificate |
| `certificate_validity_days` | int | `3650` | No | Self-signed certificate validity (10 years) |
| `letsencrypt_email` | string | - | Required if letsencrypt | Contact email for Let's Encrypt |
| `route53_zone_id` | string | - | Required if letsencrypt | AWS Route53 hosted zone ID |
| `aws_access_key_id` | string | - | Required if letsencrypt | AWS access key (from ~/.aws/credentials) |
| `aws_secret_access_key` | string | - | Required if letsencrypt | AWS secret key (from ~/.aws/credentials) |

**Example (Self-Signed)**:
```bash
ansible-playbook playbooks/setup-certificates.yml \
  -e certificate_mode=selfsigned \
  -e certificate_fqdn=registry.example.com
```

**Example (Let's Encrypt)**:
```bash
ansible-playbook playbooks/setup-certificates.yml \
  -e certificate_mode=letsencrypt \
  -e certificate_fqdn=registry.example.com \
  -e letsencrypt_email=admin@example.com \
  -e route53_zone_id=Z1234567890ABC
```

### setup-haproxy.yml

**Purpose**: Configure HAProxy load balancer with SNI routing

**Parameters**:
| Parameter | Type | Default | Required | Description |
|-----------|------|---------|----------|-------------|
| `haproxy_mode` | string | `sni` | No | Routing mode: sni, port |
| `haproxy_frontend_port` | int | `443` | No | Frontend HTTPS port (SNI mode) |
| `haproxy_stats_port` | int | `9000` | No | Stats interface port |
| `registry_backends` | list | - | Yes | List of backend registries |

**Example**:
```bash
ansible-playbook playbooks/setup-haproxy.yml \
  -e haproxy_mode=sni \
  -e '{"registry_backends":[{"fqdn":"quay.example.com","backend":"192.168.10.10:8443"}]}'
```

## Image Mirroring Playbooks

### download-to-disk-v2.yml

**Purpose**: Download OpenShift images with oc-mirror to TAR archives

**Parameters**:
| Parameter | Type | Default | Required | Description |
|-----------|------|---------|----------|-------------|
| `operator_preset_file` | string | `storage-operators` | No | Operator preset: storage-operators, networking-operators, etc. |
| `ocp_version` | string | `4.21` | No | OpenShift version to mirror |
| `ocp_channel` | string | `stable-4.21` | No | Update channel |
| `mirror_storage_path` | string | `/data/oc-mirror` | No | Directory for TAR archives |
| `oc_mirror_max_retries` | int | `3` | No | Retry failed downloads |
| `pull_secret_path` | string | `/root/pull-secret.json` | Yes | Red Hat pull secret |

**Example**:
```bash
ansible-playbook playbooks/download-to-disk-v2.yml \
  -e @extra_vars/operators/storage-operators.yml \
  -e ocp_version=4.21 \
  -e pull_secret_path=/root/rh-pull-secret.json
```

**Parameters**:

**Example**:
```bash

  -e target_registry=registry.example.com:8443 \
  -e registry_password=SecurePassword123! \
  -e source_tar_dir=/data/oc-mirror/mirror-seq1
```

## Common Parameters (All Playbooks)

### Verbosity

Control Ansible output verbosity:
```bash
ansible-playbook <playbook> -v    # Verbose
ansible-playbook <playbook> -vv   # More verbose
ansible-playbook <playbook> -vvv  # Debug mode
ansible-playbook <playbook> -vvvv # Connection debugging
```

### Check Mode (Dry Run)

Preview changes without applying:
```bash
ansible-playbook <playbook> --check
```

### Diff Mode

Show file diffs before applying:
```bash
ansible-playbook <playbook> --diff
```

### Limit to Specific Hosts

```bash
ansible-playbook <playbook> --limit registry
ansible-playbook <playbook> --limit 'registry,aap'
```

### Skip Tasks with Tags

```bash
ansible-playbook <playbook> --skip-tags provision
ansible-playbook <playbook> --skip-tags validate,test
```

## Variable Precedence

Ansible variable precedence (highest to lowest):

1. Extra vars (`-e` flag): **Highest priority**
2. Task vars
3. Block vars
4. Role vars
5. Play vars
6. Host vars (`inventory/host_vars/`)
7. Group vars (`inventory/group_vars/`)
8. Playbook vars
9. Role defaults (`roles/*/defaults/main.yml`): **Lowest priority**

**Best Practice**: Use `-e` for runtime overrides, inventory vars for environment-specific config, role defaults for sensible defaults.
