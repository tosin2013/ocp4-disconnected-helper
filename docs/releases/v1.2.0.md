## 🎯 Release Highlights

This release introduces a **comprehensive operator validation framework** that prevents wasted bandwidth by catching configuration errors before expensive mirroring operations, plus **production-validated AAP workflow orchestration** for streamlined disconnected deployments.

**Key Benefits**:
- ⚡ **99% faster error detection**: <5 seconds vs 10-30 minutes
- 💾 **99.999% bandwidth savings**: 73 KB catalog cache vs 50-100 GB full mirror
- ✅ **100% validation accuracy**: All 32 operators in 8 presets verified
- 🔄 **Production-tested AAP workflows**: Workflow Job #118 successful

## 🚀 New Features

### Operator Catalog Validation Framework (ADR-0034)

**Problem Solved**: Typos in operator names only surfaced after 10-30 minutes of expensive mirroring, wasting bandwidth and time.

**Solution**:
1. **Pre-flight Validation Playbook**:
   ```bash
   ansible-playbook playbooks/validate-operator-selection.yml \
     -e @extra_vars/operators/storage-operators.yml
   ```
   - Validates in <5 seconds (vs 10-30 min with oc-mirror)
   - Fuzzy matching suggests correct names (e.g., "local-storage" → "local-storage-operator")
   - Catalog caching (24h TTL): ~73 KB vs 50-100 GB

2. **Operator Discovery Tool**:
   ```bash
   ./scripts/discover-operators.sh --search storage
   # Returns: odf-operator, local-storage-operator, lvms-operator, etc.
   ```

3. **8 Curated Operator Presets** (`extra_vars/operators/`):
   - **storage-operators.yml**: ODF, LVMS, Local Storage, NFS, Rook Ceph (5 operators)
   - **rhacm-operators.yml**: RHACM 2.16, MCE 2.11, Submariner 0.24, GitOps (4 operators)
   - **openshift-ai-operators.yml**: RHODS, Authorino, Service Mesh, Serverless, GPU (5 operators)
   - **virtualization-operators.yml**: KubeVirt, ODF, NMState, MetalLB (4 operators)
   - **service-mesh-operators.yml**: Service Mesh, Kiali, Tempo (3 operators)
   - **observability-operators.yml**: Logging, Loki, Tempo, Observability, GitOps (5 operators)
   - **security-operators.yml**: Compliance, FIM, Quay, Quay Bridge (4 operators)
   - **networking-operators.yml**: MetalLB, NMState, Submariner, Service Mesh (4 operators)

**Validation Success Rate**: 100% (32/32 operators validated across all presets)

---

### AAP Workflow Orchestration (ADR-0032, ADR-0033)

**Problem Solved**: Manual playbook execution lacked centralized visibility, RBAC, and approval gates.

**Solution**:
1. **3-Node AAP Workflow** (Workflow ID 36: "Disconnected OpenShift Image Mirroring"):
   ```
   Node 1: Validate Operator Selection (preflight, <5s)
      ↓ ON SUCCESS
   Node 2: Download OpenShift Images (Phase 1: mirrorToDisk)
      ↓ ON SUCCESS
   Node 3: Mirror Images to Registry (Phase 2: diskToMirror)
   ```

2. **AAP Workflow Validation Framework** (ADR-0033):
   - Shell health checks: `validate-aap-health.sh`, `validate-aap-workflow-templates.sh`
   - E2E test playbooks: `test-registry-vm-workflow.yml`, `test-oc-mirror-workflow.yml`
   - GitHub Actions workflow: `validate-aap-workflows.yml`
   - Testing documentation: `docs/TESTING.md`

**Production Evidence**:
- Workflow Job #118 (June 11, 2026):
  - Validation Node: 3.6 seconds ✅
  - Download Node: 102 seconds ✅
  - Push Node: 343 seconds ✅
  - **Total**: 448.6 seconds (~7.5 minutes) ✅

## 🛠️ Migration Guide

### For Users on v1.1

No breaking changes. v1.2 is a **backwards-compatible feature release**.

**Recommended Actions**:
1. **Pull latest code**:
   ```bash
   git pull origin main
   git checkout v1.2.0
   ```

2. **Update AAP workflows** (if using AAP):
   ```bash
   ansible-playbook -i inventory/ibm-cloud.yml \
     playbooks/aap-configuration/configure-oc-mirror-workflow.yml \
     -e@extra_vars/rhel-subscription-secrets.yml \
     --vault-password-file ~/.vault_pass
   ```

3. **Try operator presets**:
   ```bash
   # Validate a preset
   ansible-playbook playbooks/validate-operator-selection.yml \
     -e @extra_vars/operators/storage-operators.yml

   # Use in mirroring
   ansible-playbook playbooks/download-to-disk-v2.yml \
     -e @extra_vars/operators/storage-operators.yml
   ```

## 🚦 Known Limitations

1. **Catalog Cache TTL**: 24 hours (may need adjustment for fast-moving operator releases)
2. **Fuzzy Matching Threshold**: 0.6 (may produce false positives on very similar names)
3. **No Auto-Refresh**: Preset operator versions are static (manual update required when Red Hat releases new versions)

**Future Work** (v1.3):
- AAP survey for preset dropdown selection
- Combination presets (e.g., "Full Platform" = Storage + Observability + Security)
- Channel upgrade path documentation
- Operator dependency graph visualization

## 📜 Installation

### Quick Start (New Installation)

```bash
# Clone repository
git clone https://github.com/tosin2013/ocp4-disconnected-helper.git
cd ocp4-disconnected-helper

# Checkout v1.2.0
git checkout v1.2.0

# Install git hooks (prevent credential commits)
./scripts/install-git-hooks.sh

# Validate operator preset
ansible-playbook playbooks/validate-operator-selection.yml \
  -e @extra_vars/operators/storage-operators.yml

# Mirror with validated operators
ansible-playbook playbooks/download-to-disk-v2.yml \
  -e @extra_vars/operators/storage-operators.yml
```

### AAP Deployment

```bash
# Deploy workflow to AAP 2.6
ansible-playbook -i inventory/ibm-cloud.yml \
  playbooks/aap-configuration/configure-oc-mirror-workflow.yml \
  -e@extra_vars/rhel-subscription-secrets.yml \
  --vault-password-file ~/.vault_pass

# Launch workflow in AAP Web UI:
# 1. Navigate to Templates → Workflows
# 2. Find "Disconnected OpenShift Image Mirroring" (ID 36)
# 3. Click "Launch"
# 4. Provide extra_vars: @extra_vars/operators/rhacm-operators.yml
# 5. Monitor execution (3 nodes, ~7-10 min total)
```
